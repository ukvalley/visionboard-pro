================================================================================
                           SYSTEM OVERVIEW
================================================================================

VisionBoard Pro is a full-stack web application built with the MERN stack
variant (MongoDB, Express, React, Node.js). It follows a modern client-server
architecture with RESTful API communication.

ARCHITECTURE TYPE:
  - Client-Server Architecture
  - RESTful API Design
  - Single Page Application (SPA)
  - MVC Pattern on Backend

================================================================================
                           TECHNICAL STACK
================================================================================

FRONTEND ARCHITECTURE:
  Framework: React 18
  Build Tool: Vite 5
  Language: JavaScript (ES6+) with JSX
  Styling: Tailwind CSS 3
  State Management: React Context API
  Routing: React Router DOM 6
  HTTP Client: Axios
  Module System: ES Modules

BACKEND ARCHITECTURE:
  Runtime: Node.js
  Framework: Express.js 4
  Language: JavaScript (CommonJS)
  Database: MongoDB
  ODM: Mongoose 8
  Authentication: JWT (jsonwebtoken)
  Password Hashing: bcryptjs
  Environment: dotenv
  CORS: cors middleware

AI INTEGRATION:
  Provider: OpenAI
  Library: openai npm package
  Features: GPT-4 for coaching, suggestions, analysis

================================================================================
                           DATA FLOW
================================================================================

CLIENT (React)                              SERVER (Express)
     |                                              |
     |  1. User Action (e.g., save section)         |
     |--------------------------------------------->|
     |         POST /api/visionboards/:id           |
     |                                              |
     |  2. Auth Middleware validates JWT          |
     |                                              |
     |  3. Controller processes request            |
     |     - Validate input                        |
     |     - Call business logic                   |
     |     - Interact with Model                   |
     |                                              |
     |  4. Model (Mongoose) interacts with DB      |
     |     - MongoDB Atlas/Local                  |
     |     - Pre-save hooks calculate progress    |
     |                                              |
     |  5. Response returned                       |
     |<---------------------------------------------|
     |     { success: true, data: {...} }         |
     |                                              |
     |  6. UI updates with new data               |
     |     React re-renders components             |

================================================================================
                           AUTHENTICATION FLOW
================================================================================

1. REGISTRATION:
   Client → POST /api/auth/register
   Server → Validate input → Hash password (bcryptjs) → Save User → Return JWT

2. LOGIN:
   Client → POST /api/auth/login
   Server → Validate credentials → Compare password → Generate JWT → Return token

3. AUTHENTICATED REQUEST:
   Client → Include JWT in Authorization header
   Server → Verify JWT (auth middleware) → Attach user to request → Proceed

4. TOKEN STRUCTURE:
   Payload: { userId, email, role, iat, exp }
   Expiration: 7 days

5. PROTECTED ROUTES:
   Frontend: ProtectedRoute component checks auth
   Backend: auth middleware validates JWT

================================================================================
                           STATE MANAGEMENT
================================================================================

CONTEXT PROVIDERS:

1. AuthContext
   - user: Current user data
   - isAuthenticated: Boolean
   - isAdmin: Boolean
   - isImpersonating: Boolean
   - impersonatedUser: Object
   - login(), logout(), impersonateUser(), stopImpersonation()

2. ThemeContext
   - theme: 'light' | 'dark'
   - toggleTheme()
   - Persists to localStorage

LOCAL STATE:
  - Component-level state with useState
  - Form data handled locally until submission

SERVER STATE:
  - Custom hooks (useVisionBoard, useAuth)
  - API calls via service layer
  - No global cache (no React Query/SWR)

================================================================================
                           SECURITY MEASURES
================================================================================

AUTHENTICATION:
  - JWT with 7-day expiration
  - Passwords hashed with bcryptjs (salt rounds: 10)
  - Protected routes on frontend and backend

AUTHORIZATION:
  - Role-based access control (user vs admin)
  - Middleware checks ownership of resources
  - Users can only access their own vision boards

INPUT VALIDATION:
  - Express-validator on backend
  - Form validation on frontend
  - MongoDB injection protection via Mongoose

CORS:
  - Configured for specific origins
  - Credentials enabled for cookies

ENVIRONMENT VARIABLES:
  - Sensitive data in .env files
  - JWT_SECRET never exposed
  - DB credentials protected

================================================================================
                           PERFORMANCE OPTIMIZATIONS
================================================================================

FRONTEND:
  - Code splitting with React.lazy()
  - Suspense for async components
  - Tailwind CSS purging unused styles
  - Vite for fast builds and HMR

BACKEND:
  - Database indexes on frequently queried fields
  - Efficient MongoDB queries
  - Error handling middleware
  - Response compression (express default)
